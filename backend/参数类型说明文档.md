## 一、类型定义说明

### 1.1 基础类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `str` | 字符串类型，用于存储文本内容 | `"这是一段文本"` |
| `Optional[str]` | 可选字符串，可以为空字符串或 None | `""` 或 `"文本内容"` |
| `bool` | 布尔类型，只有 True 或 False | `True`, `False` |
| `int` | 整数类型，用于计数、索引等 | `0`, `1`, `100` |
| `float` | 浮点数类型，用于分数、比例等 | `0.0`, `75.5`, `100.0` |
| `List[str]` | 字符串列表，存储多个文本项 | `["问题1", "问题2"]` |
| `Dict[str, Any]` | 字典类型，键值对结构，可存储多种类型数据 | `{"key": "value", "count": 10}` |

### 1.2 类型与文本传输能力

| 类型 | 能否传输文本 | 说明 |
|------|-------------|------|
| `str` | ✅ 专门用于传输文本 | 最常用的文本传输类型 |
| `Optional[str]` | ✅ 可传输文本或空值 | 适用于可选的文本参数 |
| `List[str]` | ✅ 传输多个文本项 | 适用于文本列表场景 |
| `Dict[str, Any]` | ✅ 可包含文本字段 | 字典中可存储文本内容 |
| `bool` | ❌ 不能传输文本 | 仅用于条件判断 |
| `int` | ❌ 不能传输文本 | 仅用于计数计算 |
| `float` | ❌ 不能传输文本 | 仅用于数值计算 |

## 二、工作流输入参数 (GraphInput)

工作流入口参数，用户调用时需要提供。

| 参数名 | 类型 | 必填 | 默认值 | 描述 |
|--------|------|------|--------|------|
| `reference_file` | `Optional[str]` | 条件必填 | `""` | 参考文档内容（文本格式），初始生成模式必填，续写模式可选 |
| `reference_doc` | `Optional[str]` | 否 | `""` | 参考文档内容（与 reference_file 等效，优先级更高） |
| `genre_type` | `str` | 否 | `"小说"` | 题材类型 |
| `writing_direction` | `str` | 否 | `""` | 写作内容走向描述 |
| `is_continue_writing` | `bool` | 否 | `False` | 是否为续写模式 |

### 2.1 参数详细说明

#### reference_file
- **类型**：`Optional[str]`
- **作用**：传入参考文档的文本内容，用于风格分析和内容生成
- **使用场景**：
  - 初始生成模式：必填，作为风格参考
  - 续写模式：可选，可从上下文文件恢复
- **示例**：
```json
"reference_file": "青溪村是一个与世隔绝的神秘村庄，流传着关于友谊之石的传说..."
```

#### reference_doc
- **类型**：`Optional[str]`
- **作用**：与 `reference_file` 功能相同，优先级更高
- **使用场景**：当同时传入 `reference_file` 和 `reference_doc` 时，优先使用 `reference_doc`

#### genre_type
- **类型**：`str`
- **作用**：指定写作的题材类型，影响大纲生成和内容风格
- **可选值**：
  - `小说` - 虚构故事类
  - `散文` - 抒情随笔类
  - `诗歌` - 韵律文学类
  - `剧本` - 影视戏剧类
  - `报告` - 分析报告类
  - `论文` - 学术研究类
  - `简历` - 个人简历类
  - `合同` - 法律文书类
  - `广告文案` - 营销文案类
  - `产品描述` - 产品介绍类
  - `社交媒体内容` - 新媒体内容类
  - `自定义` - 其他类型

#### writing_direction
- **类型**：`str`
- **作用**：描述写作的内容走向、情节发展方向
- **使用场景**：指导大纲生成和内容创作
- **示例**：
```json
"writing_direction": "讲述两个少年在迷雾森林中寻找友谊之石的冒险故事"
```

#### is_continue_writing
- **类型**：`bool`
- **作用**：控制工作流执行路径
- **使用场景**：
  - `False`：初始生成模式，从零开始创作
  - `True`：续写模式，基于已有内容继续创作

## 三、工作流输出参数 (GraphOutput)

工作流执行完成后返回的结果。

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `final_document` | `str` | 是 | 最终生成的文档内容（包含所有章节） |
| `style_match_report` | `str` | 是 | 风格匹配度报告（Markdown 格式） |
| `content_consistency_report` | `str` | 是 | 内容走向一致性报告（Markdown 格式） |
| `quality_report` | `str` | 是 | 内容质量评估报告（Markdown 格式） |
| `writing_progress` | `Dict[str, Any]` | 是 | 写作进度信息 |
| `final_document_file` | `Optional[str]` | 否 | 最终文档文件路径 |

### 3.1 输出参数详细说明

#### final_document
- **类型**：`str`
- **作用**：包含完整内容的最终文档
- **格式**：Markdown 格式，包含章节标题和正文
- **示例**：
```markdown
# 最终文档

题材类型：小说
总字数：8138
生成时间：2026-02-05

---
## 第一章 标题

正文内容...

## 第二章 标题

正文内容...
```

#### style_match_report
- **类型**：`str`
- **作用**：评估生成内容与参考文档的风格匹配程度
- **格式**：Markdown 格式报告
- **包含内容**：
  - 匹配度分数（0-100）
  - 词汇使用习惯分析
  - 句式结构特征分析
  - 段落组织方式分析

#### content_consistency_report
- **类型**：`str`
- **作用**：评估生成内容与写作大纲的一致性
- **格式**：Markdown 格式报告
- **包含内容**：
  - 一致性分数（0-100）
  - 情节发展符合度
  - 人物塑造一致性
  - 主题思想贯彻度

#### quality_report
- **类型**：`str`
- **作用**：综合评估生成内容的质量
- **格式**：Markdown 格式报告
- **包含内容**：
  - 质量分数（0-100）
  - 逻辑性评估
  - 连贯性评估
  - 可读性评估
  - 创新性评估

#### writing_progress
- **类型**：`Dict[str, Any]`
- **作用**：记录写作进度信息，用于续写模式
- **包含字段**：
```json
{
  "total_words": 8138,
  "genre_type": "小说",
  "writing_direction": "讲述两个少年...",
  "last_updated": "2026-02-05T10:30:00",
  "current_chapter_index": 4,
  "chapters_completed": 4,
  "batch_chapters_generated": 2,
  "total_batches": 0
}
```

#### final_document_file
- **类型**：`Optional[str]`
- **作用**：最终文档的文件存储路径
- **示例**：`"/workspace/projects/assets/final_document_final.md"`
