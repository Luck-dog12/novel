## 一、项目概述

写作风格模仿助手是一个基于 LangGraph 的工作流服务，可以基于参考文档的风格特征，根据用户提供的大纲和内容走向生成高质量文本，支持初始生成和续写两种模式。

## 四、方式一：通过 Coze API 调用（推荐）

### 4.1 获取 API Key

1. 登录 [Coze 控制台](https://www.coze.cn/)
2. 点击右上角头像 → **个人设置**
3. 选择 **API Token** 标签页
4. 点击 **生成新 Token** 或复制已有 Token

### 4.2 调用信息

| 项目 | 值 |
|------|-----|
| **请求方式** | `POST` |
| **API 地址** | `https://api.coze.cn/v1/workflow/run` |
| **Content-Type** | `application/json` |
| **认证方式** | `Authorization: Bearer <API_KEY>` |
| **超时时间** | 15分钟（900秒） |

### 4.3 CURL 示例

#### 初始生成模式

```bash
curl -X POST https://api.coze.cn/v1/workflow/run \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  --max-time 900 \
  -d '{
    "workflow_id": "7602102784212828187",
    "parameters": {
      "reference_file": "青溪村是一个与世隔绝的神秘村庄，流传着关于友谊之石的传说。据说友谊之石拥有让两个真心相待的人友谊永恒的力量。",
      "genre_type": "小说",
      "writing_direction": "讲述两个少年在迷雾森林中寻找友谊之石的冒险故事",
      "is_continue_writing": false
    }
  }'
```

#### 续写模式

```bash
curl -X POST https://api.coze.cn/v1/workflow/run \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  --max-time 900 \
  -d '{
    "workflow_id": "7602102784212828187",
```

## 六、请求参数说明

### 6.1 参数列表

| 参数名 | 类型 | 必填 | 默认值 | 描述 |
|--------|------|------|--------|------|
| `reference_file` | String | 条件必填 | `""` | 参考文档内容（文本格式），初始生成模式必填，续写模式可选 |

## 七、响应参数说明

### 7.1 响应字段

| 参数名 | 类型 | 描述 |
|--------|------|------|
| `final_document` | String | 最终生成的文档内容（Markdown格式）**⭐ 推荐使用** |
| `style_match_report` | String | 风格匹配度报告（Markdown格式） |
| `content_consistency_report` | String | 内容走向一致性报告（Markdown格式） |
| `quality_report` | String | 内容质量评估报告（Markdown格式） |
| `writing_progress` | Object | 写作进度信息 |
| `final_document_file` | String | 最终文档文件路径（外部调用无法访问） |
| `run_id` | String | 本次执行的唯一标识 |

### 7.2 writing_progress 对象结构

| 参数名 | 类型 | 描述 |
|--------|------|------|
| `total_words` | Integer | 总字数 |
| `genre_type` | String | 题材类型 |
| `writing_direction` | String | 写作内容走向 |
| `last_updated` | String | 最后更新时间 |
| `current_chapter_index` | Integer | 当前章节索引 |
| `chapters_completed` | Integer | 已完成章节数 |
| `batch_chapters_generated` | Integer | 本次生成的章节数 |
| `total_batches` | Integer | 总批次数 |

### 7.3 重要说明

⚠️ **外部调用请注意**：

| 字段 | 是否可访问 | 说明 |
|------|-----------|------|
| `final_document` | ✅ **可访问** | 文本内容直接在 JSON 响应中返回 |
| `final_document_file` | ❌ **不可访问** | 这是工作流服务器的本地文件路径 |

**请使用 `final_document` 字段获取文章内容！**

## 八、HTTP 状态码说明

| 状态码 | 含义 | 说明 |
|--------|------|------|
| `200` | ✅ 成功 | 请求成功，返回生成的文章内容 |
| `400` | ❌ 请求错误 | 参数格式错误或缺少必填参数 |
| `401` | ❌ 认证失败 | API Key 无效或过期（方式一） |
| `500` | ❌ 服务器错误 | 工作流执行过程中发生异常 |
| `504` | ❌ 网关超时 | 请求超时，可尝试重试 |

## 九、Python 调用示例

```python
import requests

def call_writing_assistant(payload: dict, api_key: str = None) -> dict:
    """
    调用写作风格模仿助手工作流
    
    Args:
        payload: 请求参数
        api_key: API Key（可选，不传则使用项目域名调用）
    
    Returns:
        dict: 响应结果
    """
    if api_key:
        # 方式一：Coze API 调用
        url = "https://api.coze.cn/v1/workflow/run"
        headers = {
            "Authorization": f"Bearer {api_key}",
            "Content-Type": "application/json"
        }
        body = {
            "workflow_id": "7602102784212828187",
            "parameters": payload
        }
    else:
        # 方式二：项目域名调用
        url = "https://433c4ef6-4399-4953-9946-5a356f124dbc.dev.coze.site/run"
        headers = {"Content-Type": "application/json"}
        body = payload
    
    response = requests.post(url, json=body, headers=headers, timeout=900)
    
    if response.status_code == 200:
        result = response.json()
        if "final_document" in result and result["final_document"]:
            return result
        else:
            return {"error": "内容为空"}
    else:
        return {"error": f"状态码 {response.status_code}", "detail": response.text}


# 使用示例
if __name__ == "__main__":
    # 初始生成模式
    result = call_writing_assistant(
        payload={
            "reference_file": "参考文档内容...",
            "genre_type": "小说",
            "writing_direction": "故事走向",
            "is_continue_writing": False
        },
        api_key="YOUR_API_KEY"  # 可选
    )
    
    # 获取文章内容
    article = result.get("final_document", "")
    print(article)
```

## 十、常见问题

### Q1: API Key 和项目域名两种方式有什么区别？

| 对比项 | Coze API（方式一） | 项目域名（方式二） |
|-------|-------------------|-------------------|
| **认证** | 需要 API Key | 无需认证 |
| **安全性** | 高，可追踪调用来源 | 低，任何人可调用 |
| **稳定性** | 高，官方维护 | 依赖项目部署状态 |
| **适用场景** | 正式环境、生产调用 | 测试、内部调用 |

### Q2: 如何获取 API Key？

1. 登录 Coze 控制台
2. 个人设置 → API Token
3. 生成或复制已有 Token

### Q3: 续写模式如何使用？

续写模式需要先执行一次初始生成，生成的内容会保存在上下文中。后续调用时设置 `is_continue_writing: true` 即可继续生成新章节。
