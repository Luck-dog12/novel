# 数据模型设计：安卓小说写作助手应用

**Created**: 2026-02-03  
**Based On**: [Implementation Plan](./plan.md)

## 核心实体

### 1. 用户配置 (UserConfig)

**描述**: 存储用户的应用配置和偏好设置

**属性**:
- `id`: 唯一标识符
- `theme`: 主题设置 (亮色/暗色)
- `fontSize`: 字体大小
- `fontFamily`: 字体家族
- `autoSave`: 自动保存设置
- `notificationEnabled`: 通知设置
- `lastUpdated`: 最后更新时间

**关系**:
- 与用户是一对一关系

### 2. 写作项目 (WritingProject)

**描述**: 存储用户的写作项目信息

**属性**:
- `id`: 唯一标识符
- `title`: 项目标题
- `description`: 项目描述
- `genreType`: 题材类型
- `creationDate`: 创建日期
- `lastModifiedDate`: 最后修改日期
- `status`: 项目状态 (进行中/已完成)
- `wordCount`: 字数统计

**关系**:
- 与用户是多对一关系
- 与生成历史是一对多关系
- 与参考文档是一对多关系

### 3. 生成历史 (GenerationHistory)

**描述**: 存储历史生成记录

**属性**:
- `id`: 唯一标识符
- `projectId`: 关联的写作项目ID
- `generationType`: 生成类型 (初始生成/续写)
- `inputParams`: 输入参数 (JSON格式)
- `outputContent`: 生成内容
- `generationDate`: 生成日期
- `duration`: 生成耗时 (毫秒)

**关系**:
- 与写作项目是多对一关系

### 4. 参考文档 (ReferenceDocument)

**描述**: 存储用户上传的参考文档信息

**属性**:
- `id`: 唯一标识符
- `projectId`: 关联的写作项目ID
- `fileName`: 文件名
- `fileType`: 文件类型 (md/txt)
- `fileSize`: 文件大小 (字节)
- `uploadDate`: 上传日期
- `filePath`: 本地存储路径
- `contentHash`: 内容哈希值 (用于检测变化)

**关系**:
- 与写作项目是多对一关系

### 5. 上下文信息 (ContextInfo)

**描述**: 存储写作上下文，用于后续生成

**属性**:
- `id`: 唯一标识符
- `projectId`: 关联的写作项目ID
- `contextType`: 上下文类型
- `contextContent`: 上下文内容
- `lastUpdated`: 最后更新时间

**关系**:
- 与写作项目是多对一关系

## 数据流程

### 前端数据流程
1. **用户配置**:
   - 应用启动时加载用户配置
   - 用户修改配置时保存到本地存储

2. **写作项目**:
   - 显示项目列表
   - 创建新项目
   - 编辑现有项目
   - 删除项目

3. **参考文档**:
   - 上传文档
   - 预览文档内容
   - 管理文档列表

4. **生成请求**:
   - 收集用户输入参数
   - 发送请求到后端服务
   - 显示加载状态
   - 接收并展示生成结果
   - 保存生成历史

5. **本地缓存**:
   - 缓存生成结果
   - 缓存参考文档内容
   - 缓存用户操作历史

### 后端数据流程
1. **请求处理**:
   - 接收前端请求
   - 验证请求参数
   - 调用Coze API
   - 处理API响应
   - 返回结果给前端

2. **Coze API集成**:
   - 构建API请求
   - 处理认证
   - 发送请求到Coze
   - 处理响应和错误
   - 转换数据格式

3. **数据存储**:
   - 存储请求日志
   - 存储API响应缓存
   - 存储用户使用统计

4. **错误处理**:
   - 捕获网络错误
   - 处理API错误
   - 记录错误日志
   - 返回友好错误信息

## 数据验证规则

### 1. 用户配置验证
- `theme`: 必须是预定义的值 (light/dark)
- `fontSize`: 必须在合理范围内 (8-24)
- `fontFamily`: 必须是可用的字体

### 2. 写作项目验证
- `title`: 不能为空，长度不能超过100个字符
- `genreType`: 必须是预定义的值
- `status`: 必须是预定义的值 (in_progress/completed)

### 3. 参考文档验证
- `fileName`: 不能为空
- `fileType`: 必须是md或txt
- `fileSize`: 不能超过20MB

### 4. 生成请求验证
- 初始生成模式必须提供参考文档或题材类型
- 续写模式必须提供参考文档或参考内容
- 所有必填参数必须非空

## 存储策略

### 前端存储
- **用户配置**: 使用 SharedPreferences 存储
- **写作项目**: 使用 Room 数据库存储
- **生成历史**: 使用 Room 数据库存储
- **参考文档**: 存储为本地文件，元数据使用 Room 数据库存储
- **上下文信息**: 使用 Room 数据库存储
- **缓存数据**: 使用内存缓存和磁盘缓存

### 后端存储
- **请求日志**: 使用结构化日志存储
- **API响应缓存**: 使用 Redis 存储
- **用户使用统计**: 使用数据库存储

## 数据安全

### 前端安全
- **本地存储加密**: 对敏感数据进行加密存储
- **文件权限**: 确保参考文档只能被应用访问
- **输入验证**: 验证所有用户输入，防止注入攻击

### 后端安全
- **API密钥保护**: 安全存储Coze API密钥
- **请求验证**: 验证所有前端请求，防止未授权访问
- **数据传输加密**: 使用HTTPS加密传输数据
- **错误处理**: 避免在错误响应中泄露敏感信息
- **日志脱敏**: 对日志中的敏感信息进行脱敏处理

## 扩展性考虑

### 数据模型扩展
- **预留字段**: 为核心实体预留扩展字段
- **模块化设计**: 采用模块化设计，便于添加新功能
- **版本控制**: 实现数据模型版本控制，支持迁移

### 性能优化
- **索引设计**: 为常用查询字段创建索引
- **批量操作**: 支持批量数据操作，减少数据库交互
- **延迟加载**: 对大型数据采用延迟加载策略
- **缓存策略**: 实现多级缓存，提高数据访问速度

### 兼容性考虑
- **向后兼容**: 确保数据模型变更向后兼容
- **数据迁移**: 提供数据迁移工具和脚本
- **跨平台兼容**: 考虑数据结构在不同平台的兼容性
