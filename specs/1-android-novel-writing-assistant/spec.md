# Feature Specification: 安卓小说写作助手应用需求分析与规划

**Feature Branch**: `1-android-novel-writing-assistant`  
**Created**: 2026-02-03  
**Status**: Draft  
**Input**: User description: "针对安卓小说写作助手应用进行需求分析与规划。该应用需基于现有Coze工作流实现核心功能，要求用户交互界面与工作流试运行功能保持一致。具体任务包括：1.分析提供的Coze工作流数据流向节点图和框架构成图，梳理核心功能模块与数据处理逻辑；2.解析工作流试运行输入参数内容，明确用户交互所需的输入项与格式要求；3.规划安卓应用的用户界面结构，确保界面元素与工作流功能一一对应；4.制定需求分析文档，包含功能需求、用户界面需求、数据流程需求及技术实现约束。当前阶段仅进行需求分析，不涉及具体技术实现细节。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 初始生成模式 (Priority: P1)

用户选择初始生成模式，上传参考文档，选择题材类型，描述写作内容走向，系统基于这些输入生成小说内容。

**Why this priority**: 初始生成是应用的核心功能之一，是用户首次使用应用时的主要流程。

**Independent Test**: 用户可以通过上传文档、选择题材、填写写作方向并启动生成来测试此功能。

**Acceptance Scenarios**:

1. **Given** 用户打开应用，**When** 选择初始生成模式，上传参考文档，选择题材类型，填写写作内容走向，点击生成按钮，**Then** 系统开始处理并生成小说内容。
2. **Given** 用户在初始生成过程中，**When** 系统完成处理，**Then** 用户可以查看生成的小说内容。

### User Story 2 - 续写模式 (Priority: P1)

用户选择续写模式，系统基于同一写作会话（sessionId / context_version）自动加载此前写作产生的上下文（例如：题材、写作方向、大纲、人物设定、世界观、剧情大纲、章节梗概、上一章节内容等），用户无需再次上传参考文档或输入参考内容，系统基于上下文生成续写。

**Why this priority**: 续写模式是应用的核心功能之一，满足用户对现有内容进行扩展的需求。

**Independent Test**: 用户先完成一次初始生成（产生章节内容并保存上下文），再进入续写模式直接启动生成来测试此功能。

**Acceptance Scenarios**:

1. **Given** 用户已完成一次初始生成且系统已生成写作会话标识（sessionId）并保存上下文，**When** 用户进入续写模式并点击生成按钮，**Then** 系统使用该sessionId读取同一链路上下文并生成续写内容。
2. **Given** 用户进入续写模式，**When** 系统未找到任何可用上下文，**Then** 系统提示用户先完成一次初始生成后再进行续写。
3. **Given** 用户在续写过程中，**When** 系统完成处理，**Then** 用户可以查看生成的续写内容。

### User Story 3 - 工作流参数配置 (Priority: P2)

用户可以配置工作流的输入参数，包括参考文档、题材类型、写作内容走向等。

**Why this priority**: 参数配置是生成和续写功能的基础，确保用户可以根据需求定制输入。

**Independent Test**: 用户可以通过填写不同的参数组合来测试系统对参数的处理能力。

**Acceptance Scenarios**:

1. **Given** 用户进入参数配置界面，**When** 填写各项参数，**Then** 系统保存参数并用于后续处理。
2. **Given** 用户填写了无效参数，**When** 点击生成按钮，**Then** 系统提示参数错误并要求修正。

### Edge Cases

- 当用户上传过大的参考文档时，系统应给出适当的提示。
- 当用户未填写必填参数时，系统应在点击生成按钮时提示用户。
- 当网络连接中断时，系统应保存用户的输入并在网络恢复后继续处理。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须支持初始生成模式，允许用户上传参考文档、选择题材类型、填写写作内容走向。
- **FR-002**: 系统必须支持续写模式；续写模式必须基于已保存的写作上下文进行生成，用户无需再次上传参考文档或输入参考内容。
- **FR-003**: 系统必须能够分析用户输入的参数，并根据写作模式执行相应的工作流。
- **FR-004**: 系统必须能够展示生成的小说内容，包括初始生成和续写结果。
- **FR-005**: 系统必须能够处理文档上传功能，支持md和txt格式，单个文件大小不超过20MB。
- **FR-006**: 系统必须能够保存写作上下文信息，用于后续的内容生成；上下文至少包含“上一章节内容”，并应在初始生成后自动生成并保存题材、写作方向、大纲、人物设定、世界观、剧情大纲、章节梗概、章节名称等结构化信息。
- **FR-007**: 系统必须为每次初始生成创建写作会话标识（sessionId / context_version）并返回给客户端；客户端必须保存该标识并在续写请求中携带，系统基于该标识读取同一链路的上下文，确保多实例并发下续写上下文准确一致。

### Key Entities

- **参考文档**: 用户上传的md或txt格式文件，用于提供写作参考。
- **题材类型**: 小说、散文、诗歌、剧本、报告、论文、简历、合同、广告文案、产品描述、社交媒体内容、自定义等。
- **写作内容走向**: 用户对写作内容的描述，指导生成方向。
- **生成内容**: 系统根据用户输入生成的小说内容。
- **上下文信息**: 系统保存的写作上下文，用于后续生成。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户可以在5分钟内完成初始生成或续写的参数配置并启动生成。
- **SC-002**: 系统能够正确处理95%以上的用户输入参数。
- **SC-003**: 90%的用户能够成功完成初始生成或续写任务。
- **SC-004**: 系统界面与工作流功能的对应度达到95%以上。

## Additional Analysis

### Coze工作流分析

根据提供的工作流数据流向节点图，系统包含以下核心模块：

1. **文档上传与内容识别**: 处理用户上传的参考文档，识别文档内容。
2. **写作模式判断**: 根据用户选择判断是初始生成还是续写模式。
3. **初始生成流程**:
   - 题材类型选择与要求确认
   - 写作大纲生成
   - 风格分析
   - 大纲细化
   - 内容生成
   - 生成并初始化结构化上下文（题材、写作方向、大纲、人物设定、世界观、剧情大纲等）
   - 保存上下文（章节名称、章节梗概、上一章节内容等）
4. **续写模式流程**:
   - 续写循环
5. **文档格式化与智能性生成**: 对生成的内容进行格式化和智能处理。

### 工作流试运行参数分析

根据提供的试运行界面，系统需要以下输入参数：

1. **reference_file**: 参考文档文件（md或txt格式），初始生成模式必填。
2. **genre_type**: 题材类型，包括小说、散文、诗歌、剧本、报告、论文、简历、合同、广告文案、产品描述、社交媒体内容、自定义等。
3. **writing_direction**: 写作内容走向描述。
4. **is_continue_writing**: 是否为续写模式，布尔值；续写模式下系统从已保存上下文中读取所需信息，不要求用户再次提供reference_file/reference_doc等参考输入。
5. **session_id**: 写作会话标识；初始生成时由系统生成并返回，续写时必须携带，用于从同一链路的上下文中读取所需信息。

### 安卓应用界面规划

基于工作流分析，安卓应用界面应包含以下部分：

1. **主界面**:
   - 应用标题
   - 初始生成模式入口
   - 续写模式入口
   - 历史记录入口

2. **初始生成模式界面**:
   - 参考文档上传区域
   - 题材类型选择器
   - 写作内容走向输入框
   - 生成按钮

3. **续写模式界面**:
   - 上下文加载提示与预览（例如：上一章节内容）
   - 生成按钮

4. **处理状态界面**:
   - 处理进度显示
   - 取消按钮

5. **结果展示界面**:
   - 生成内容展示
   - 保存按钮
   - 分享按钮
   - 返回按钮

### 技术实现约束

1. **基于现有Coze工作流**: 应用必须基于现有的Coze工作流实现核心功能，确保功能一致性。
2. **用户界面与工作流对应**: 界面元素必须与工作流功能一一对应，确保用户操作流畅。
3. **参数格式要求**: 必须严格按照工作流试运行的参数格式要求实现输入功能。
4. **文档处理限制**: 单个参考文档大小不得超过20MB，支持md和txt格式。
5. **无技术实现细节**: 当前阶段仅进行需求分析，不涉及具体技术实现细节。
